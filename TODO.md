# New tool ideas

## generateTool

Lets the agent generate their own custom MCP tool that persists during the session.

### toolbox

Maybe have the tools generated by the generateTool persist and make a toolbox tool that lets the agent see all of the tools that it has created so it can restore tools dynamically and have those tools register themselves when the toolbox tool is invoked so the agent can keep that tool loaded into its context as a directly invokable tool.

This is a context optimization feature that would allow the agent to create and manage specific tools that it generates without touching the MCP server's source code directly or having to restart the MCP server or rebuild anything for the new tool to become available.

## generateApp

A tool that allows the agent to generate JSX that will render in the MCP tool's iframe output for the user to visualize and interact with. Kind of like how Claude's native chat feature works for generating interactive JSX for the user to explore ideas interactively without having to build a dedicated MCP tool for every type of app.

## calculator

An advanced calculator tool that allows the agent to do advanced calculations using a variety of supported mathematical formulas and equation visualization features via an MCP app.

## listDirectories

A tool that lets you list directories with variable depth, a gitignore bool, etc.
